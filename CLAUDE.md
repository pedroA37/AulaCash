## Project Overview

Educational virtual wallet simulation (no real money). Inspired by Mercado Pago / BNA MODO. Full-stack: Node.js/Express backend + React/Vite PWA frontend. The UI must always clarify it is a non-regulated educational project.

## Commands

### Backend (`cd backend`)
```bash
npm run dev       # development with nodemon
npm start         # production
npm run migrate   # apply migrations/001_initial.sql to DB
npm run seed      # create first admin user (uses ADMIN_EMAIL / ADMIN_PASSWORD env vars)
```

### Frontend (`cd frontend`)
```bash
npm run dev       # Vite dev server on :5173
npm run build     # production PWA build
npm run lint      # ESLint
npm run preview   # preview production build
```

## Architecture

### Backend

```
backend/src/
├── server.js           # Express app: cors, helmet, JSON body, route mounting, error handler
├── config/db.js        # pg Pool singleton via DATABASE_URL; SSL auto-enabled in production
├── middleware/auth.js  # requireAuth (JWT verify), requireAdmin (JWT + rol === 'admin')
├── routes/             # thin routers — only mount middleware and call controllers
├── controllers/        # request/response logic; SQL lives here directly (no ORM)
├── services/
│   └── generadores.js  # generarAliasUnico(), generarCBUUnico() — retry loops against DB
```

Route groups:
- `POST /api/auth/*` — registro, login, forgot-password, reset-password
- `GET|POST|PATCH /api/cuenta/*` — me, buscar, alias, email, transferencias, transacciones, qr/*
- `GET|POST|PATCH|DELETE /api/admin/*` — protected by `requireAdmin`

### Atomic Money Operations

Both `transferir` (transferenciaController) and `cobrarQR` (qrController) use the same pattern — **never bypass it**:

```js
const client = await pool.connect();
await client.query('BEGIN');
// SELECT ... FOR UPDATE on all affected rows (prevents race conditions)
// validate saldo, estado, expiry, self-transfer
// UPDATE saldo - origen, + destino
// INSERT transacciones
await client.query('COMMIT');
client.release();
// catch: ROLLBACK, rethrow; finally: client.release()
```

The `CHECK (saldo >= 0)` DB constraint is the last line of defense.

### QR Flow

1. Emisor calls `POST /api/cuenta/qr/generar` → creates `codigos_qr` row (UUID token, 30-min expiry, estado=`pendiente`).
2. Emisor displays QR encoding the URL `/pagar/:token`.
3. Pagador scans, frontend GETs `/api/cuenta/qr/:token` to preview amount/recipient.
4. Pagador confirms → `POST /api/cuenta/qr/cobrar` → atomic transfer, marks QR `usado`.

QR states: `pendiente` → `usado` | `expirado`.

### Frontend

```
frontend/src/
├── main.jsx            # React root, mounts <App />
├── App.jsx             # BrowserRouter + route guards (RutaPrivada, RutaAdmin, RutaPublica)
├── context/AuthContext.jsx  # JWT in localStorage; exposes { usuario, cargando, login, logout, setUsuario }
├── services/api.js     # Axios instance; request interceptor attaches Bearer token;
│                       # response interceptor clears token + redirects to /login on 401
├── pages/              # one file per route
└── components/         # Layout (wraps pages with BottomNav), BottomNav, Spinner, Comprobante
```

Auth state is bootstrapped by calling `GET /api/cuenta/me` on mount if a token exists in localStorage.

Route guards in `App.jsx`:
- `RutaPrivada` — redirects to `/login` if unauthenticated
- `RutaAdmin` — redirects to `/dashboard` if not admin
- `RutaPublica` — redirects to `/dashboard` (or saved `redirect_after_login` from sessionStorage) if already authenticated

### Database Schema

Four tables: `usuarios`, `transacciones`, `codigos_qr`, `password_resets`.

Key constraints to keep in mind:
- `usuarios.saldo CHECK (saldo >= 0)` — enforced at DB level
- `transacciones.usuario_origen_id` is nullable (NULL = carga_admin)
- `transacciones.tipo` ∈ `('transferencia', 'carga_admin', 'cobro_qr')`
- `codigos_qr.token` is UUID generated by `gen_random_uuid()` (requires `pgcrypto` extension)
- Password reset tokens expire in 1h, `usado` flag prevents reuse
